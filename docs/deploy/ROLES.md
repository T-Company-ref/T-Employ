# 직원 역할 · 알림 설정

## 역할 이름: **추천자** (`recommender`)

지원자·인재 상세에서 **추천하기**만 가능하고, 블락(삭제)·단계 변경·면접 등록은 불가합니다.

| 역할 (DB) | 표시명 | 조회 | 추천 | 블락/단계/면접 |
|-----------|--------|------|------|----------------|
| `operator` | 운영자 | ✓ | ✓ | ✓ |
| `recruiter` | 채용담당 | ✓ | ✓ | ✓ |
| `recommender` | **추천자** | ✓ | ✓ | — |
| `viewer` | 조회자 | ✓ | — | — |

추가 추천자 등록 시 `staff_profiles.role = 'recommender'` 로 넣으면 됩니다.

## 추천 태그 표시

이력서 아래 **추천하기** → `candidate_tags` (`tag_type=recommend`)  
칩에 **별명(`nickname`)** 이 함께 표시됩니다. 예: `추천 (hj.joo)`

## 프로필·알림 설정

**모든 역할**(운영자·채용담당·추천자·조회자)이 상단 오른쪽 **이름**을 눌러 본인 설정을 바꿀 수 있습니다.

- 표시 이름 (`display_name`)
- 별명 (`nickname`) — 추천 태그에 사용
- 메일 알림 (`notify_pref`)

| `notify_pref` | 의미 |
|---------------|------|
| `none` | 알림 안 받음 |
| `digest` | 아침 다이제스트만 (~07:37) |
| `realtime` | 실시간 지원 알림 + 다이제스트 |

운영 장애·인증 오류 메일은 계속 `ACTION_NOTIFY_EMAIL`(기본 `yj.kim@tbell.co.kr`)로 갑니다.
